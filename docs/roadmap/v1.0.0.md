# v1.0.0 — General Availability

**Status:** Planned
**Target audience:** All Windows 10/11 users
**Prerequisite:** v0.4.0 (public beta) successfully closed

---

## What v1.0.0 Means

v1.0.0 is the **General Availability** release of ArizenOS. It signals:

- The core user experience is production-ready for everyday use on primary machines
- `@arizen/agent-sdk` and `@arizen/widgets` APIs are stable with a formal SemVer commitment
- An LTS (Long-Term Support) program is active
- The governance model transitions to include a community-elected Technical Council
- Enterprise support programs are available

v1.0.0 is not a feature freeze — ArizenOS will continue evolving. It is a quality and stability certification.

---

## Deliverables

### New Capabilities

#### ARM64 Support (Snapdragon X Elite / X Plus)
- [ ] All N-API native modules compiled for ARM64
- [ ] llama.cpp ARM64 backend with NEON SIMD optimisation
- [ ] Glass rendering via DirectComposition on ARM64 Windows
- [ ] CI matrix updated to include ARM64 runners
- [ ] Installation tested on Surface Pro X, Snapdragon X machines

#### ArizenOS AI Runtime (Public API)
An optional local HTTP server that exposes Arizen Assistant and ArizenMind as an OpenAI-compatible API endpoint on localhost. Enables third-party applications to use ArizenOS as their local AI backend.

- [ ] OpenAI-compatible `/v1/chat/completions` endpoint
- [ ] Streaming support
- [ ] Authentication: local API key (no external auth)
- [ ] Configurable in Hub → Settings → AI Runtime
- [ ] Disabled by default (explicit opt-in)

#### Enterprise Deployment Support
- [ ] Silent installer mode (MSI + transforms)
- [ ] Group Policy (GPO) support for managed configuration
- [ ] Managed theme deployment (organisation themes pushed via policy)
- [ ] Audit log export (SIEM-compatible JSON)
- [ ] Enterprise Hub: private extension repository

### Governance

#### Arizen Foundation
- [ ] Legal entity established as steward of the ArizenOS project
- [ ] Technical Council elections (5 seats, elected by contributor community)
- [ ] Foundation governance documents ratified
- [ ] IP assignment from Arizen Technologies to the Foundation
- [ ] Trademark policy for community forks and ecosystem

#### LTS Program

ArizenOS v1.0.0 starts a formal LTS cycle:

| Release | Type | Support Until |
|---|---|---|
| v1.0.0 | LTS | 3 years from release |
| v1.x.x | Patch releases | Ongoing during LTS window |
| v2.0.0 | Next major (when ready) | TBD |

LTS releases receive:
- Security patches (within 14 days of disclosure)
- Critical bug fixes
- No new features (feature work goes to the next major version branch)

### Quality Targets

| Target | Requirement |
|---|---|
| Crash rate | < 1 crash per 500 hours of usage |
| Shell memory (idle) | < 200 MB RAM |
| Startup time | < 3 seconds from Windows login to fully interactive shell |
| GPU frame time | < 2ms at High glass quality on GTX 1660 |
| AI TTFT (Phi-3 Mini) | < 500ms time-to-first-token on RTX 3060 |
| AI TTFT (CPU-only) | < 3000ms on Intel Core i5-12400 |

| Security | Annual penetration test by independent firm |
| Test coverage | `@arizen/core`: 100%, others: ≥ 80% |

## What v1.0.0 Is NOT

- **Not a macOS or Linux port.** Cross-platform is not in the v1.0.0 scope.
- **Not a Windows replacement.** ArizenOS remains an experience layer — not a separate OS.
- **Not a cloud product.** Cloud AI remains opt-in. The core is and will remain local-first.
- **Not feature-complete forever.** v1.0.0 is GA, not end-of-life. v2.0.0 planning begins after GA.

---

## Release Criteria

All of the following must be true before v1.0.0 ships:

1. All v0.4.0 quality gates pass on v1.0.0-rc builds
2. ARM64 builds pass all tests on physical Snapdragon X hardware
3. LTS governance documents are ratified by the Technical Council
4. Security penetration test complete with all Critical/High findings resolved
5. `@arizen/agent-sdk` and `@arizen/widgets` APIs reviewed and approved by the Technical Council
6. Documentation site fully live at [docs.arizenos.dev](https://docs.arizenos.dev)
7. Enterprise deployment tested by at least 3 organisations in a pilot program
