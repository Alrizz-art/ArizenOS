## Community Support

### GitHub Discussions *(Preferred)*

For questions, troubleshooting, and how-to guidance — use **[GitHub Discussions](https://github.com/Alrizz-art/ArizenOS/discussions)**.

Select the appropriate category:
- **Q&A** — "How do I…?" questions
- **Ideas** — Feature suggestions (early-stage)
- **Show and Tell** — Extensions, themes, setups you've built
- **General** — Everything else

## Bug Reports

If you've confirmed a bug (not a question or configuration issue), open a **[Bug Report](https://github.com/Alrizz-art/ArizenOS/issues/new?template=bug_report.yml)** on GitHub.

Please include:
- ArizenOS version (`arizen --version`)
- Windows version (`winver`)
- GPU model and driver version
- Steps to reproduce (exact sequence)
- Expected vs. actual behaviour
- Logs if available (`%LOCALAPPDATA%\ArizenOS\logs\`)

Reports without reproduction steps will be closed as `needs-info`.

---

## Feature Requests

For small features, open a **[Feature Request](https://github.com/Alrizz-art/ArizenOS/issues/new?template=feature_request.yml)** issue.

For significant changes that affect architecture, APIs, or public interfaces, please open an **[RFC](https://github.com/Alrizz-art/ArizenOS/issues/new?template=rfc.yml)** first. This lets the community discuss the design before implementation begins.

---

## Security Vulnerabilities

**Do not open a public issue for security vulnerabilities.**

Report them privately via:
- [GitHub Private Vulnerability Reporting](https://github.com/Alrizz-art/ArizenOS/security/advisories/new)
- Email: [security@arizenos.dev](mailto:security@arizenos.dev)

See [SECURITY.md](SECURITY.md) for our full disclosure policy.

---

## Enterprise / Commercial Support

Enterprise support programs are planned for the v1.0.0 release. If you are evaluating ArizenOS for enterprise deployment prior to v1.0.0, contact [enterprise@arizenos.dev](mailto:enterprise@arizenos.dev).

---

## What We Don't Support Here

- Third-party extensions and themes (contact their authors)
- Unofficial forks or modified builds
- Custom Windows configurations that conflict with ArizenOS requirements
- Issues on Windows versions below the [minimum requirements](README.md#-installation)

## Response Time Expectations

| Channel | Expected Response |
|---|---|
| GitHub Discussions | 1–5 business days |
| Discord `#help` | Best-effort, community-driven |
| Bug Reports (tagged `confirmed`) | Triaged within 7 days |
| Security Reports | Acknowledged within 48 hours |

> Response times are best-effort. ArizenOS is in pre-alpha and maintained by a small core team and volunteers.
