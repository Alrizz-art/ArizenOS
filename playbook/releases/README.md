# playbook/releases/ — Release Artifacts and Metadata

> **Purpose:** Release manifests, changelogs, and checksums for all shipped versions  
> **Ownership:** Core Team  
> **Release Process:** Updated as part of the pre-release checklist (see RELEASE_STRATEGY.md)

---

## Purpose

`playbook/releases/` is the release record layer. It does not contain the `.apbx` files (those are GitHub Release assets) but contains all metadata and documentation for every release.

This directory is the canonical source of truth for:
- What was in each release
- When it shipped
- What Windows versions it supports
- Its SHA256 checksum
- Its changelog

---

## Directory Structure

```
playbook/releases/
├── README.md
├── manifests/
│   ├── latest.json              ← Pointer to current stable release
│   └── v{version}.json          ← Metadata for each release
├── changelogs/
│   └── v{version}.md            ← Human-readable changelog per version
└── checksums/
    └── v{version}.sha256        ← SHA256 of the .apbx for each version
```

## Checksum File Format (`checksums/v{version}.sha256`)

Standard `sha256sum` output format:

```
{64-char-sha256-hash}  ArizenOS.apbx
```

Generated by:
```powershell
$hash = Get-FileHash ArizenOS.apbx -Algorithm SHA256
"$($hash.Hash.ToLower())  ArizenOS.apbx" | Out-File playbook/releases/checksums/v0.1.0.sha256 -Encoding ascii
```

Users can verify their downloaded `.apbx`:
```powershell
$hash = Get-FileHash .\ArizenOS.apbx -Algorithm SHA256
$expected = (Get-Content .\v0.1.0.sha256).Split(" ")[0]
if ($hash.Hash.ToLower() -eq $expected) { "✓ Checksum valid" } else { "✗ CHECKSUM MISMATCH" }
```
